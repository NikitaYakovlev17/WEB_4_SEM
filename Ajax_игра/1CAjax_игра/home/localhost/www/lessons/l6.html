Кроме этого, запрос нужно отправить, используя тип POST. Если передать параметры в составе url, то они не будут использоваться! Ах да! Запрос нужно выполнить синхронно, иначе борт-компьюетр обработает пустой результат! Удачи! Хм. Но ведь чтобы передать параметры в запросе с типом POST нужно использовать ... Обычно, запросы с типом GET используются для получения данных, а тип POST - для изменения или передачи данных. По сравнению с GET, POST-запрос может иметь неограниченный размер содержимого(размер данных), в то время как длина URL ограничена как для GET, так и для POST запросов.
Для отправки данных в запросе с типом POST необходимо следующее:
  • Указать первым параметром в методе open тип "POST", далее указать путь и асинхронность
  • Установить заголовок "Content-type" в "application/x-www-form-urlencoded" используя метод setRequestHeader: xhttp.setRequestHeader("Content-type", "application/x-www-form-urlencoded");
  • Вызвать метод send с набором параметром как ключ=значение, объединенные между собой символом &
Сформируйте правильный POST запрос, который выполнится синхронно.
